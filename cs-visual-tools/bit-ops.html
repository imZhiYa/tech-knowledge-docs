💡 为什么八进制与十六进制是资深架构师与设计师的绝佳工具？

纯二进制序列漫长单调，肉眼极易看错。因为 2³ = 8，2⁴ = 16，3 个比特刚好浓缩为 1 位八进制（如 Linux 权限 chmod 0755）；4 个比特刚好组成 1 个十六进制字（1 Byte = 2 个 Hex 字符，如 Java 魔数 0xCAFEBABE 或 Web 设计颜色 #FF3366）。

public boolean hasPrivilege(long target) {
// 1. 🟢 探查鉴权：高速 O(1) 判定 (&)

}

public void grantPrivilege(long target) {
// 2. 🔵 赋能加权：安全合并注入 (|=，天然幂等)
this.userBundle |= target;
}

public void revokePrivilege(long target) {
// 3. 🟣 定向抹除：精准卸下标记不波及周边 (& ~)
this.userBundle &= ~target;
}

🛡️ 为什么不用德摩根定律直接写 ~ (flags | mask) != 0？

在 WAF 防火墙验证中，许多老手也会踩坑：试图判断“多个特权是否同时不存在”，写成 (~(flags | mask)) != 0。

❌ 灾难陷阱：按位取反 ~ 会将无关的高位空虚零全部翻转为 1！这会导致 != 0 永恒成立，造成全网正常请求被 WAF 当场误杀！

✅ 最佳工程共识：最严谨的规范是合并掩码做归零断言：(flags & (MASK_A | MASK_B)) == 0。彻底杜绝脏数据！
